Here's an overview of the files and structure in the bolt-python repository:

## Repository Structure:

### Root Files:
- `README.md` - Documentation (11 KB)
- `LICENSE` - License file
- `pyproject.toml` - Python project configuration
- `setup.cfg` - Setup configuration
- `.flake8` - Linting configuration
- `codecov.yml` - Code coverage configuration
- `MANIFEST.in` - Package manifest

### Main Directories:

1. **`slack_bolt/`** - Core library source code with subdirectories:
   - `adapter/` - Framework adapters (Flask, FastAPI, Django, etc.)
   - `app/` - Main application code
   - `authorization/` - Authorization handling
   - `context/` - Context management
   - `oauth/` - OAuth implementation
   - `middleware/` - Middleware components
   - `listener/` - Event listener logic
   - `request/`, `response/` - HTTP handling
   - `workflows/` - Workflow functionality
   - And more...

2. **`examples/`** - 26 example subdirectories including:
   - Framework integrations: `flask/`, `django/`, `fastapi/`, `sanic/`, `starlette/`, `tornado/`, etc.
   - Cloud platforms: `aws_lambda/`, `aws_chalice/`, `google_cloud_functions/`, `google_app_engine/`, `heroku/`
   - `docker/` - Docker examples
   - `assistants/` - AI assistants examples
   - `getting_started/` - Beginner examples
   - Various other integration examples

3. **`tests/`** - Test suites:
   - `slack_bolt/` - Synchronous tests
   - `slack_bolt_async/` - Async tests
   - `adapter_tests/` & `adapter_tests_async/` - Adapter tests
   - `scenario_tests/` - Integration tests

4. **`docs/`** - Documentation in English and Japanese

5. **`.github/`** - GitHub workflows and issue templates

The repository contains **1,024 non-git files** organized across these directories, primarily consisting of Python source code, examples, tests, and documentation.
