## Bolt Python Adapters

The adapter directory contains **16 different web framework adapters** that allow Bolt apps to run on various Python web frameworks and platforms:

### üåê **Web Framework Adapters:**

1. **`flask/`** - Flask integration (sync)
2. **`fastapi/`** - FastAPI integration (async)
3. **`django/`** - Django integration
4. **`sanic/`** - Sanic async framework
5. **`starlette/`** - Starlette async framework
6. **`tornado/`** - Tornado framework (both sync & async)
7. **`falcon/`** - Falcon REST framework
8. **`bottle/`** - Bottle micro-framework
9. **`pyramid/`** - Pyramid web framework
10. **`cherrypy/`** - CherryPy framework
11. **`aiohttp/`** - AIOHTTP async framework

### ‚òÅÔ∏è **Cloud Platform Adapters:**

12. **`aws_lambda/`** - AWS Lambda serverless (includes Chalice support)
    - Handler for Lambda functions
    - Chalice framework handler
    - S3-based OAuth flow
    - Lazy listener runners
13. **`google_cloud_functions/`** - Google Cloud Functions

### üîå **Protocol Adapters:**

14. **`socket_mode/`** - WebSocket-based Socket Mode (no HTTP endpoint needed)
    - Multiple implementations: `builtin`, `aiohttp`, `websockets`, `websocket_client`
    - Both sync and async handlers
15. **`asgi/`** - Generic ASGI adapter (for any ASGI-compatible framework)
    - Builtin and AIOHTTP implementations
16. **`wsgi/`** - Generic WSGI adapter (for any WSGI-compatible framework)

### üìù **Adapter Structure:**

Each adapter typically contains:
- `__init__.py` - Exports the handler class
- `handler.py` or `async_handler.py` - Main handler implementation
- Some have additional support files for OAuth, lazy listeners, etc.

This comprehensive adapter collection means you can run Bolt apps on **virtually any Python web framework or cloud platform**!
