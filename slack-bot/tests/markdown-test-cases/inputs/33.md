## Key Files & Root Directory Overview

### **Root Directory Structure:**

```

├── Configuration Files
│   ├── pyproject.toml          # Main package config (build, dependencies, tools)
│   ├── setup.cfg               # Legacy package metadata (URL, author)
│   ├── .flake8                 # Linting config (max-line-length: 125)
│   ├── codecov.yml             # Code coverage settings (50% patch target)
│   ├── .gitignore              # Git ignore patterns
│   └── MANIFEST.in             # Package distribution files
│
├── Documentation
│   ├── README.md               # Main documentation
│   ├── LICENSE                 # MIT License
│   └── docs/                   # Full documentation
│
├── Source Code
│   └── slack_bolt/             # Main package (18 modules)
│
├── Testing & Examples
│   ├── tests/                  # Test suite
│   ├── examples/               # 20+ example apps
│   └── requirements/           # Dependency files
│       ├── testing.txt         # Test dependencies
│       ├── async.txt           # Async dependencies
│       ├── adapter.txt         # Web framework adapters
│       └── tools.txt           # Dev tools
│
├── Development Tools
│   ├── scripts/                # Build & deployment scripts
│   │   ├── run_tests.sh
│   │   ├── format.sh
│   │   ├── lint.sh
│   │   ├── run_mypy.sh
│   │   └── build_pypi_package.sh
│   └── logs/                   # Test logs directory
│
└── GitHub Workflows
    └── .github/
        ├── workflows/          # CI/CD pipelines
        │   ├── ci-build.yml
        │   ├── pypi-release.yml
        │   └── triage-issues.yml
        ├── contributing.md     # Contribution guidelines
        ├── maintainers_guide.md
        └── CODEOWNERS
```

### **Key Files:**

**Configuration & Build:**
- `pyproject.toml` - Modern Python packaging config, build system, testing, linting rules
- `.flake8` - Code style rules (125 char line length)
- `codecov.yml` - Code coverage thresholds

**Core Source Files:**
- `slack_bolt/__init__.py` - Main exports (App, Context, Listeners, AI/Assistant classes)
- `slack_bolt/app/app.py` - Core synchronous App class
- `slack_bolt/async_app.py` - Asynchronous App class
- `slack_bolt/version.py` - Version number (1.27.0)

**Package Structure Highlights:**
- **17 web framework adapters** (Flask, Django, FastAPI, AWS Lambda, etc.)
- **Middleware system** (auth, request verification, SSL check, URL verification)
- **AI/Assistant support** with thread context management
- **OAuth implementation** built-in
- **Context utilities** (ack, say, respond, complete, fail)

The project is well-organized with clear separation between source code, tests, examples, and tooling!
